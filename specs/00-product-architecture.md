# 00 Product Architecture

## goal

定义 TuBu 徒步装备助手 V1 MVP 的产品边界、页面结构和核心闭环，为后续实现阶段提供统一方向。

## scope

- 产品定位：个人徒步、露营、轻装/重装出行的装备管理与出行清单规划小程序。
- MVP 闭环：新增装备 -> 创建方案 -> 从装备库加入方案 -> 临时新增消耗品 -> 自动统计重量 -> 分类可视化 -> 打包检查 -> 复制方案复用。
- 底部 Tab：首页、装备、新增、方案、我的。
- 核心对象：装备库、出行方案、方案装备清单。

## data impact

- 本阶段不实现数据读写。
- 后续数据模型必须包含 `Gear`、`TripPlan`、`PlanItem`。
- `PlanItem` 必须保存装备快照，装备库变更不得影响历史方案。

## UI impact

- 首页展示装备库总览、默认方案、当前方案统计和快捷入口。
- 装备页管理长期装备库。
- 新增页提供手动新增和本地智能解析入口。
- 方案页管理出行方案。
- 我的页第一期只做轻量设置/占位，不承载核心功能。

## acceptance criteria

- 能说明第一期只围绕三个问题：拥有哪些装备、这次带哪些装备、重量和打包状态。
- 页面结构不包含社区、商城、会员、KOL、路线内容。
- 后续 sprint 可以明确映射到某个阶段 spec。
- 微信开发者工具中第一屏应优先呈现核心统计，而不是营销或社区内容。

## out of scope

- 真实 AI 推荐。
- 天气联动。
- 社区、活动、KOL、商城、会员。
- 多用户协作。
- 云开发或后端接口实现。
